Core principle:

Build the following capabilities:

1. SEO PRESERVATION ENGINE
Create a preservation workflow that runs before any build is finalized.

It must:

- ingest Ahrefs exports if provided
- support future ingestion of Google Search Console exports

- a “Redirect Map” table

2. URL PRESERVATION RULES
Add strict logic so the build engine does not casually change URLs.

Rules:
- keep existing URLs wherever possible
- if a URL changes, a 301 redirect must be created
- never redirect important pages to the homepage unless there is no relevant equivalent
- preserve slug intent, not just page existence
- preserve high-value service pages, location pages, and blog pages if they carry rankings, backlinks, or internal authority

I want this logic enforced in the build flow, not just displayed as advice.

3. PAGE INTENT PROTECTION
For every important page, preserve and improve its ranking intent.

The system should evaluate:
- what service the page is about
- what location intent exists
- whether it appears to support commercial, transactional, or informational search intent
- whether it aligns with a GBP service or service area

Do not allow a strong page to be replaced by a generic page that weakens intent.

Example:
If an existing page ranks for emergency electrician in a target suburb, the rebuild must preserve that intent with an equivalent or better page.

- map website service pages to GBP services

- support consistent NAP and business identity placement

- prevent the site from drifting away from GBP categories, services, and geography
- flag inconsistencies between planned website structure and GBP strategy

- GBP alignment score
- GBP consistency checklist

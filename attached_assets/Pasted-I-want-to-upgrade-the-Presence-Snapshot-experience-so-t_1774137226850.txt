I want to upgrade the Presence Snapshot experience so the summary stays clean, but each meaningful item can be clicked to inspect deeper evidence in a modal.

Goal:
Turn the Presence Snapshot from a flat list of technical findings into an interactive “scan + inspect” experience. The main card should stay short and readable, while deeper explanation and evidence live in a modal.

What to build:
1. Make Presence Snapshot rows clickable when supporting drilldown data exists
2. Open a modal (or right-side drawer if that fits the design system better) when the user clicks a row
3. Use plain-English labels in the main card
4. In the modal, show:
- what we found
- why it matters
- recommended improvement
- supporting evidence
- optional technical details

UX requirements:
- Keep the collapsed snapshot compact and scannable
- Do not overload the main card with long explanations
- Only make rows clickable if real supporting detail exists
- If there is no drilldown data, render the row as static text
- Use product-native language, not raw SEO jargon where possible
- Preserve the premium feel of the Focus View / AI panels

Main card row examples:

Modal structure:
For each clicked insight, show sections in this order:
1. Summary
2. Why it matters
3. Evidence
4. Recommended improvement
5. Technical details (optional / collapsible)

Suggested data shape:
Create a reusable insight detail model so Presence Snapshot can support multiple insight types consistently.

Example TypeScript shape:

type PresenceInsightDetail = {
id: string;
label: string;
status: "positive" | "warning" | "neutral";
summary: string;
whyItMatters?: string;
recommendedImprovement?: string;

value: string;

type PresenceSnapshotItem = {

Interaction rules:
- If item.detail exists and has meaningful evidence or explanation, row is clickable
- Clicking row opens PresenceInsightModal
- Modal title should use human-readable language
- Add hover/cursor states only for clickable rows
- Include a small chevron, external-inspect icon, or subtle “View details” affordance for clickable items
- Keyboard accessible: Enter/Space should open if focused
- Escape closes modal
- Follow existing shadcn/ui + Radix patterns already used in the app

Build an AI-driven “Next Best Action” (NBA) system that links the Action Queue (located in the Daily Planner page) to Pipeline leads/deals (located on a separate Pipeline page), recommends the highest-leverage next step using AI + deterministic rules, and lets the rep execute the action from a phone (tap-to-call, tap-to-text, tap-to-email) with suggested copy prefilled. All actions must log back into Activities, Deal/Lead History, and Dashboard counters. Add a “Why this action” expand panel to each queue row and add a Focus Mode that locks the rep into the top 3 actions until done.

Action Queue is displayed/managed on the Daily Planner page.

Deals/Leads are managed on the Pipeline page.

This separation must NOT break linking or logging.
Requirements:

Every Action Queue item must store targetType + targetId pointing to a Pipeline record.

Use a URL param pattern such as:

#/pipeline?openType=deal&openId={targetId}

#/pipeline?openType=lead&openId={targetId}

auto-expand/open that card

scroll it into view

Stage changes or activity logs done on the Pipeline page must immediately affect NBA ranking and Action Queue ordering on the Daily Planner page (real-time subscription preferred).

AI RECOMMENDATION ENGINE (SERVER-SIDE)
Create a server-side AI function (Cloud Function/backend) that recommends the single best next action for a given lead/deal. Do not expose API key client-side.

AI INPUTS MUST INCLUDE

nowLocal + timezone

rep dailyTargets and todaySoFar (calls/meetings/proposals)

lead/deal summary: businessName, contactName, stage, estimatedValue, nextContactDate, lastContactAt, nepqLabel, notes, doNotCall, preferredContactMethod, phone/email availability

last 10 activities (type + timestamp + notes)

last 10 history events (stage changes + major actions)

optional pipeline health summary (counts per stage)

AI OUTPUT MUST BE STRICT JSON ONLY (no markdown/text)
Schema:
{
"suggestedActionType": "call"|"sms"|"email"|"meeting"|"dropin"|"proposal"|"followup"|"research",

Rules:

suggestedEmail populated only if suggestedActionType == "email" (otherwise null/omit).

Provide exactly 3 NEPQ questions.

Do not invent facts; missing info → choose “research” and list missingFields.

HYBRID SCORING (RULES + AI)
Compute baseline priorityScore with deterministic rules; allow AI to adjust within 0–100.
Required rules:

Overdue follow-up: now > nextContactDate AND no activity since → +30

Stage urgency:

Proposal Sent and no follow-up in 2 days → +25

Contacted and no activity in 3 days → +20

Value weighting: higher value → up to +15

Stagnation: same stage > 7 days → +15

Activity balance:

behind call target → prefer call unless doNotCall

behind meeting target → prefer meeting-setting actions

Points mapping:

call 5, sms 3, email 3, meeting 8, dropin 8, proposal 6, followup 5, research 2 (or 0)

Caching:

Cache per targetId for 30–60 minutes unless record/activity/history changes.

ACTION QUEUE POPULATION & AUTO-REFRESH

On Daily Planner load and on relevant data changes (lead/deal updates, activity write, stage change, history write), generate/update queue.

Maintain top 10 open items sorted by priorityScore DESC then dueAt ASC (nulls last).

Only generate actions for:

Prevent duplicates using fingerprint and/or “one open per targetId” rule.

Build a Daily Plan module in BattleScore that is AI-managed, activity-driven, and enforces Fanatical Prospecting discipline, while balancing new prospecting and current client actions.

The Daily Plan must remove guesswork and tell the user exactly what to do today, in what order, and why.

1) PURPOSE (NON-NEGOTIABLE)

The Daily Plan exists to:

enforce time-blocked execution

ensure daily prospecting never stops

ensure current clients are actively managed

convert intention into measurable activity

close the gap between planning and execution

The AI Agent is responsible for building, prioritising, and adjusting the plan.

2) DAILY PLAN PAGE STRUCTURE

The Daily Plan page contains these sections, in this order:

AI-Generated Daily Plan Summary

Daily Targets (Prospecting + Clients)

Time Blocks (AI-Allocated)

Route Plan (Door Knocking / Drop-Ins)

Today’s Action Queue

End-of-Day Debrief (AI-Reviewed)

The plan is generated once per day and can be regenerated on demand.

3) AI-GENERATED DAILY PLAN SUMMARY (TOP OF PAGE)

At the top of the page, show an AI Agent summary:

The AI Agent must:

review pipeline leads

review nurture leads

review current clients

review overdue follow-ups

review today’s calendar (if available)

review yesterday’s results

Output a short plan summary:

“Today’s focus”

“Non-negotiable actions”

“Risk areas if skipped”

This section is read-only and sets intent for the day.

4) DAILY TARGETS (ACTIVITY-BASED)

Targets are split into Prospecting and Current Clients.

Prospecting Targets

Calls

Door knocks / drop-ins

New conversations started

Meetings booked

Current Client Targets

Client check-ins

Upsell / cross-sell conversations

Retention or renewal actions

Outstanding client follow-ups

Targets default from settings but are AI-adjustable daily based on workload.

Display:

Target vs completed

BattleScore earned per category

5) AI-ALLOCATED TIME BLOCKS (CORE)

The AI Agent must build the day using time blocks.

Each block includes:

Block name

Start time

End time

Block type:

Prospecting (Doors)

Prospecting (Calls)

Client Management

Meetings

Admin (optional)

Activity target for the block

Rules:

At least one Prospecting block per day is mandatory

Prospecting blocks cannot be replaced by client work

Client work is scheduled after prospecting unless explicitly overridden

If no Prospecting block exists, show warning:

“Prospecting is not scheduled. Fanatical Prospecting requires protected time blocks.”

6) ROUTE PLAN (AI-ASSISTED, DISCIPLINE-FIRST)

The Route Plan lives inside a Prospecting (Doors/Drop-Ins) block.

AI responsibilities:

Recommend which prospects to include

Prioritise:

Least recently contacted

Highest priority score

Territory clustering

Ensure route supports minimum activity target

Required inputs:

Start location (default: Home Base)

Prospecting block selection

Average stop duration

Minimum stops target

AI must warn if:

route exceeds realistic capacity

activity target cannot be hit

7) ROUTE OPTIMISATION LOGIC

Optimisation priorities:

Hit minimum activity volume

Maintain prospecting mix (new + follow-ups)

Minimise drive time

Implementation:

Google Routes / Directions API preferred

Fallback heuristic if unavailable

Store:

Ordered stops

Estimated drive time

Estimated finish time

8) TODAY’S ACTION QUEUE (SINGLE SOURCE OF TRUTH)

Create a unified Today’s Action Queue, generated by the AI Agent.

It must include:

Prospecting actions (calls, doors)

Client actions (check-ins, follow-ups, renewals)

Meetings prep

Overdue items

Sorted by:

Time block

Urgency

Priority score

Each item has:

Action type

Lead/client

One-click execution

One-click completion logging

9) EXECUTION LOGGING (REAL-TIME)

When an action is completed:

Log activity

Increment relevant target

Add BattleScore points

Prompt for next action if applicable

No manual double-entry.

10) END-OF-DAY DEBRIEF (AI-REVIEWED)

At day end:

Planned vs completed

Prospecting completed vs skipped

Client actions completed vs skipped

BattleScore earned

AI Agent must:

identify gaps

suggest improvements

generate tomorrow’s focus

Day cannot be closed without debrief.

11) FANATICAL PROSPECTING PRINCIPLES (ENFORCED)

Prospecting is scheduled first

Activity before outcomes

Volume before optimisation

Daily accountability loop

No zero-prospecting days

12) BUILD ORDER (DAILY PLAN ONLY)

Daily Plan layout

AI plan generation logic

Targets (prospecting + clients)

Time block enforcement

Route Plan integration

Action Queue

Debrief + AI review

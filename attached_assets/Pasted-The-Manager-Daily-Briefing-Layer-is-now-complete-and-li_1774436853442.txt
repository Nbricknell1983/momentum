Next, I want to build the Unified Cross-System Operations View.

Context:
- Momentum now handles lead intelligence, strategy presentation, proposal acceptance, onboarding transition, sales execution, client portal access, Agent Command, Expansion Engine, Cadence + Automation, Communication Drafting, Executive Reporting, Automation Execution, Referral Engine, Autopilot Policy, Scheduled Sweeps, Real Provider Sending, and Daily Briefing

GOAL
Build the Unified Cross-System Operations View so managers/operators can:
- see sales-side and delivery-side state together
- understand handoff flow from lead → proposal → onboarding → provisioning → delivery → optimisation
- identify cross-system bottlenecks
- spot blocked or stalled accounts across the full lifecycle
- see where intervention is needed right now
- operate the whole business from one command surface

IMPORTANT
This must be:
- operator-focused
- high-signal
- premium in UX
- explainable
- cross-system
- action-oriented

DO NOT:
- duplicate every detailed workspace
- expose raw internal logs by default
- make it a generic BI dashboard
- rebuild AI Systems admin inside Momentum
- flatten both systems into an unreadable wall of widgets

- tenant provisioned

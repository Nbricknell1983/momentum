Build Brief — Public Strategy Page + Acceptance, Using the Existing Strategy Structure
Goal
Turn the existing public strategy page into a first-class commercial artifact that:

preserves the current structure and layout exactly as the foundation
is populated automatically from the strategy object
helps win the deal
captures acceptance
converts accepted scope into real delivery work in Bullpen
Core product decision
Do not redesign the public strategy page from scratch.

The current public strategy page structure is already the right foundation and should be preserved.

The work now is to:

keep the current layout hierarchy
make the content generation and acceptance workflow robust
connect acceptance to delivery

What needs to be built
Phase 1 — Canonical strategy object → current page structure mapping
Create or finalize a structured strategy object that maps cleanly into the existing page layout.

The strategy object needs to drive:
hero strategy direction
growth readiness score + sub-scores
triangle evidence
search opportunity data
buyer journey interpretation
reality gap language
intent coverage rows
audit cards
roadmap
pillars
snapshots
cost of inaction
projected outcomes
acceptance scope
Important
Do not let page sections become disconnected ad hoc outputs.
The page should render from one coherent strategy data object.

Phase 2 — Strengthen content generation for the current sections
The agents need to populate the existing layout with strong content, not placeholders.

The content needs to feel:
commercially sharp
easy to discuss on a sales/follow-up call
persuasive without sounding generic
grounded in what is known plus reasonable interpretation
Especially important sections to strengthen
Strategy Direction
Momentum Moment
Buyer Reality Gap
Growth Roadmap
Growth Pillars
Cost of Inaction
Scope Acceptance framing
These are the sections most likely to influence the close.

Phase 3 — Make the page a first-class public URL
The page should be publishable as a public strategy link with the existing layout preserved.

Requirements
shareable public URL
clean rendering across devices
commercially credible presentation
optional PDF export from same strategy object if needed later
no structure drift between internal and public versions
Important
The public page should be the web-native version of the existing strategy structure.
Not a separate parallel artifact.

Phase 4 — Acceptance capture on the existing acceptance section
Use the existing Scope Acceptance Section as the operational capture point.

Required behavior
The bottom section should:

capture selected services
capture contact name/email/notes
store acceptance state
support accepted/not accepted state
show accepted services after submission
Better support if practical
partial acceptance
notes/scope modifiers
follow-up status
Phase 5 — Accepted scope → delivery conversion
When a prospect accepts services from the acceptance section, the system should automatically convert them into real Bullpen delivery work.

Required outputs
work items

implementation-ready context from the strategy
onboarding/delivery initiation where needed
Commercial rules
1. This page must help win the work
It is not just a diagnostic report.
It should:

sharpen belief
reduce uncertainty
frame opportunity
make the recommended scope feel logical
2. Acceptance must feel natural
The acceptance section should not feel bolted on.
It should feel like the natural next step after the strategy.

3. Accepted strategy must become delivery automatically
Do not let accepted scope die as passive history.

4. Keep the current strategy structure
No redesign drift.
Improve the engine, not the skeleton.

Recommended implementation phases
Phase 1 — Strategy object mapping
define mapping from current strategy data to every existing page section
identify missing fields/derived values needed to populate all sections well
Phase 2 — Public page content hardening
strengthen page copy and section logic using current structure
ensure strategy generation produces all necessary section content
Phase 3 — Acceptance capture
make the current acceptance section store real accepted scope and notes cleanly
Phase 4 — Delivery conversion
convert accepted services into Bullpen work items and specialist ownership automatically

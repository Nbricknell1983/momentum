ebsite Workstream (Agent + UI)

Context
We’re building a “Website Workstream” in Momentum (React/Node/Firestore app). The goal: agents synthesize existing intelligence (Website X-Ray, SERP, Strategy, Growth Prescription) into a structured Website Blueprint for a client, and the UI lets a manager review, tweak copy choices, accept, and export/publish. All runs are persisted (snapshot + engineHistory) and respect our job queue (TTL, idempotency, retries).

What to deliver

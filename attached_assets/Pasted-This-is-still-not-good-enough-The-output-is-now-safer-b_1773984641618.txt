That is not acceptable for a business with clearly discoverable public presence.

What I need fixed now:

1. Presence Snapshot must actively discover obvious public presence
It should not rely only on fields already stored on the lead.

For a normal business, the system should actively attempt to discover:
- primary website
- sitemap
- crawlable page/content presence
- social profiles
- GBP / Maps presence
- competitor signals where supported

2. Website discovery must improve
For First Parking, the system should be able to find:
- https://www.firstparking.com.au/
- sitemap / sitemap_index.xml
- page/content presence

Implement or improve active website discovery so Presence Snapshot does not remain “not yet verified” when an obvious company website exists.

3. Social discovery must improve
The system should not wait for facebookUrl/linkedinUrl to already exist in Firestore if those are publicly discoverable.

At minimum, inspect whether existing crawler/search/enrichment logic can:
- extract outbound social links from the website
- detect obvious public social profiles from search/business signals
- populate stored social fields or a presence snapshot layer accordingly

For this business, LinkedIn and Facebook should be discoverable.

4. GBP / Maps discovery must improve
“Not yet verified” is still too weak here.

Inspect and improve the current GBP / Maps lookup path so the system can actively identify likely GBP presence for established businesses, especially where multiple locations may exist.

Do not just wait for a pre-linked Place ID if public business presence can be detected more intelligently.

For obvious public data like website/social/basic GBP presence, the system should find it itself.

6. Search visibility reasoning must improve
Do not say visibility is low because website/GBP are “unverified” when the system simply failed to discover obvious assets.
Fix the upstream discovery so search visibility reasoning is based on actual signals.

7. This must be a general fix, not a one-off patch
Do not hardcode First Parking.
Improve the active discovery pipeline used by Presence Snapshot.

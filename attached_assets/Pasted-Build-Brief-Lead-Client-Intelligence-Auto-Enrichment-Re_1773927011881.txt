Build Brief — Lead & Client Intelligence Auto-Enrichment, Research Completion & Dependency Signalling
Goal
Make Bullpen’s agents automatically enrich all leads and clients by researching and populating missing intelligence fields where possible, while explicitly flagging any missing dependencies, integrations, or paid data resources required to complete the rest.

Core outcome
Every lead and client record should move toward being:

more complete
more useful
more researched
more operationally ready
without relying on Nathan to manually fill obvious fields one by one.

If the agents cannot complete something, they should not leave silent blanks.
They should explain why and what is needed to unlock it.

Problem
Momentum now has:

Bullpen threads
work queue
specialist routing
trigger engine
daily reviews
approvals
delivery review
But a lot of lead/client intelligence still depends on:

manual input
one-off scans
disconnected enrichment actions
empty fields with no explanation
That means the records are not as operationally rich as they could be.

The agents should be doing the work of:

researching
inferring
enriching
flagging blockers
keeping records current
across all leads and clients, not just on-demand one at a time.

Outcome that matters most
Make lead and client records proactively richer and more actionable.

Success means:

missing fields are auto-filled where confidence is high
public research is used where appropriate
strategic sections are generated and refreshed
blanks are explainable
dependency/resource requirements are surfaced explicitly
all active leads and clients can be processed systematically
Scope
In scope
intelligence field coverage map across leads and clients
automated enrichment/research passes for all leads and clients
confidence-aware field population
dependency/resource signalling for blocked fields
enrichment status visibility per record
scheduling / batch processing model for all active records
Out of scope
guaranteed completion of every field regardless of available data
blind low-confidence autofill
immediate integration with every external paid data source
full custom ETL platform
rewriting the CRM data model from scratch
Required architecture rules
1. No silent blanks
For each important field, the system should do one of:

populate it automatically
mark it as suggested / pending review
mark it as not found
mark it as blocked by missing dependency/resource
A blank with no explanation is not acceptable once the enrichment system runs.

2. Confidence must govern write behavior
High confidence → auto-fill
Medium confidence → suggestion/review flag if appropriate
Low confidence → do not write as truth; create note/finding instead
3. Use specialist ownership by field domain
Do not let one generic enrichment prompt try to do everything equally well.

4. Batch safely across all leads and clients
This should work at scale:

all active leads
all active clients
without noisy duplication, repeated junk writes, or uncontrolled API cost
5. Missing resources must become explicit work
If enrichment is blocked by missing APIs/integrations/tools, create:

dependency note
work item
blocker
rather than silently skipping it

Phase 4 — Dependency / resource signalling
If a field cannot be filled because a resource is missing, make it explicit.

Examples
Ahrefs/API not connected
GBP OAuth missing
Local Falcon not available
no sitemap found
no reliable social profile found
website unreachable
external lookup quota/tool unavailable
Required behavior
Create one of:

enrichment blocker on the record
Bullpen work item
dependency note
status badge on the field cluster
Example message
“SEO backlink and keyword intelligence unavailable — Ahrefs integration/API required.”

That is the behavior you asked for.

Phase 6 — Queue/work item integration
Use Bullpen’s work system where the enrichment process finds:

Important

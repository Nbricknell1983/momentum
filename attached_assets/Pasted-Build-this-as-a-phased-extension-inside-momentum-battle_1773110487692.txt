Build this as a phased extension inside momentum.battlescore.com.au.

This is not a redesign of the platform. It is an additive sales execution layer for Momentum that helps reps prepare for calls, handle objections, follow up faster, generate more prospects, stay focused daily, and identify stalled deals before they are lost.

The goal is to turn Momentum from a pipeline tracker into a sales execution platform.

1. Product objective

Create a new sales intelligence and execution layer inside Momentum with four connected capabilities:

AI Sales Engine

Live Call Intelligence

Momentum Dashboard

Deal Momentum Score

These features must work together, but should be built in phases to reduce risk, protect the current architecture, and avoid unnecessary rework.

2. Problem we are solving

Reps currently have to think across too many disconnected tasks:

who to contact

how to prepare

what to say

how to handle objections

how to follow up

how to create more pipeline

which deals are actually moving versus stalling

This creates hesitation, inconsistent follow-up, poor pipeline hygiene, and missed sales opportunities.

Momentum should solve this by giving the rep a guided workflow directly inside the pipeline.

3. In scope

Build the following inside momentum.battlescore.com.au only:

A. AI Sales Engine

A right-side lead-aware panel that supports:

Win Before You Dial

Control the Call

Win the Follow-Up

Multiply Your Pipeline

B. Live Call Intelligence

A live mode within AI Sales Engine that helps during calls using typed call notes initially.

C. Momentum Dashboard

A daily sales dashboard that shows:

Momentum Score

Today’s Mission

AI Next Best Actions

Pipeline Health

Momentum Streak

Prospect Generator

Weekly Wins

D. Deal Momentum Score

A score on each lead and in lead detail that indicates deal health:

Strong

Active

At Risk

Stalled

4. Out of scope

Do not:

redesign the entire Momentum product

move this into the CRM deal page

rewrite the lead panel architecture

rewrite kanban pipeline logic

change auth

change tenant resolution

affect public websites

affect booking widgets

affect quotes, invoices, jobs, or marketing site modules

auto-send email or SMS without explicit user action

introduce cross-tenant prompt history or AI output leakage

This must remain a focused extension of Momentum only.

5. Tenant and architecture guardrails

This is critical.

All AI outputs, scores, dashboard data, and lead actions must remain strictly scoped by:

tenantId

leadId

userId where relevant

Do not allow:

cross-tenant access

shared prompt history between tenants

shared dashboard metrics between tenants

lead suggestions or scores leaking between orgs

Preserve existing working functionality. Prefer extending existing patterns rather than creating parallel or duplicate systems.

Phase 4 — Deal Momentum Score
Objective

Show which deals are progressing and which are stalling.

This must appear:

on each lead card

in the selected lead panel

in the Momentum Dashboard as a prioritisation signal

V1 scoring model

Base score: 50

Add:

call logged in last 3 days: +10

meeting logged in last 5 days: +15

follow-up sent in last 3 days: +8

next action scheduled: +10

stage moved in last 7 days: +15

proposal sent in last 7 days: +12

Subtract:

no activity in 7+ days: -15

no next action set: -12

overdue next action: -15

same stage for 10+ days: -10

proposal sent with no follow-up in 5+ days: -12

multiple touches without reply: -8

Clamp score between 0 and 100.

Status bands

80–100 = Strong

60–79 = Active

35–59 = At Risk

0–34 = Stalled

UI requirements
On lead cards

Add a small badge or pill, for example:

82 Strong

61 Active

34 At Risk

12 Stalled

Do not let this dominate the card.

In selected lead panel

Add a section near the top:

Deal Momentum Score

Show:

score out of 100

status label

2–3 reasons for current score

suggested next step

Example reasons:

No contact logged in 9 days

No next step scheduled

Proposal sent 6 days ago with no follow-up

In dashboard

Add sections for:

lowest momentum deals

stalled deals needing attention

Data approach

Prefer one of these:

compute on read from existing lead activity

cache on lead document for performance

Suggested cached fields on lead:

momentumScore

momentumStatus

momentumReasons

momentumUpdatedAt

Guardrails

do not create unnecessary duplicate systems

do not slow down board rendering

preserve drag/drop and lead card interactions

keep score computation tenant-scoped

Acceptance criteria

each lead card shows a score badge

selected lead panel shows score, reasons, next step

score updates when activity changes

dashboard can surface low-momentum deals

no regression to board performance or usability

6. Design direction

This should feel like premium SaaS sales tooling, not a novelty AI add-on.

Visual rules

Use:

structured cards

subtle borders

clean typography

compact spacing

clear section hierarchy

strong scanability

Avoid:

chatbot bubbles

noisy gradients

gimmicky AI visuals

anything that disrupts current Momentum usability

Interaction rules

support empty state, loading state, retry state, and error state

keep interactions fast

preserve keyboard and click flow where possible

wrap long AI outputs correctly

mobile/smaller laptop should degrade cleanly into drawer/slide-over patterns

Before implementation, review these carefully:

shared Momentum layout components

kanban board rendering and drag/drop

selected lead panel state management

notes save flow

any feature flags or admin gating patterns already in place

Do not rewrite tightly coupled areas unless absolutely necessary. Extend existing patterns.

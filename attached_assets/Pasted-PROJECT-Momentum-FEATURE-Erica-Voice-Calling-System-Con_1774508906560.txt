PROJECT: Momentum
FEATURE: Erica Voice Calling System (Context-Aware, Human-Controlled, Vapi-Powered)

IMPORTANT CONTEXT

This is the Momentum project.

Momentum owns:
- lead intelligence
- deal intelligence
- client intelligence
- sales execution
- proposal follow-up
- nurture workflows
- cadence
- communication logic
- expansion / churn / referral logic
- onboarding transition
- operator controls

This is NOT the AI Systems website/runtime project.

Do NOT build tenant website voice assistants here.
Do NOT build generic uncontrolled auto-dialing here.

Erica is a Momentum voice agent.

--------------------------------------------------
CORE PRODUCT GOAL
--------------------------------------------------

Build Erica as a controlled, context-aware voice calling system inside Momentum.

Her job is to:
- sell the appointment
- book nurture/review calls
- qualify leads
- follow up strategically
- handle objections naturally
- write outcomes back into Momentum

She must use:
- NEPQ principles
- Chris Voss / Never Split the Difference style framing
- SWISH / pattern interruption techniques where appropriate

But she must NEVER operate from a generic script.

She must always call with:
- the correct call context
- the correct deal/client context
- the correct objective
- the correct opening angle

--------------------------------------------------
CRITICAL RULES
--------------------------------------------------

1. Erica must NEVER build her own calling list
A human must manually select the companies/leads/accounts first.

2. Erica must NEVER call without a call brief
Before every call she must receive:
- Deal Intelligence
- Client Intelligence (if relevant)
- source section
- call objective
- objection predictions
- question plan
- close strategy

3. Erica must behave differently based on WHERE the call was launched from

Examples:
- Prospects / Leads = cold / semi-cold call
- Clients = nurture / review / expansion call
- Cadence = follow-up call
- Expansion = upsell call
- Referral = referral ask call

4. Vapi is the voice layer only
Momentum remains the source of truth for:
- call preparation
- logic
- context
- allowed actions
- policy
- outcomes

5. Erica must remain human-controlled
No uncontrolled auto-calling.
No autonomous list building.
No bulk calling without an explicit launch action.

Build the full Erica Calling System inside Momentum.

If source = clients:
- callType = nurture

- objective = book_review_call

If source = cadence:
- callType = follow_up
- relationship = depends on entity
- objective = continue current process

If source = expansion:
- callType = upsell
- objective = explore growth opportunity

If source = referral:
- callType = referral_ask
- objective = request introduction/referral

Create a structured EricaCallContext object that is attached to every batch item and later passed into Vapi.

9. ERICA CONVERSATION FRAMEWORK

Build Erica’s logic so she uses:
- NEPQ discovery
- Voss labeling/mirroring/calibrated questions
- SWISH-style reframe/pattern interruption where appropriate

But this must be grounded in the call brief.

Examples:
- cold lead → permission-based opening, discovery, problem awareness, appointment close
- nurture client → relationship-based opening, reference history, review call booking
- upsell call → value-led check-in, identify gap, position review call
- referral ask → leverage win/momentum, soft ask or direct intro ask based on context

Do not use one generic opener for every call.

10. OPENING STRATEGY RULES

Examples:

- ask permission

proposal_follow_up:

- identify blocker

nurture:
- reference relationship/history
- provide value
- book review/nurture call

upsell:
- reference results/opportunity
- position conversation as growth review

referral_ask:
- reference win / positive momentum
- ask in the correct style based on readiness

Momentum — Client Pipeline Workspace + AI Client Growth Engine + Public Reports
Objective

Extend the existing Sales Pipeline workspace architecture into the Client Pipeline so that client records open using the same full workspace layout, but with client growth intelligence instead of deal intelligence.

The client pipeline should become an AI Account Manager / AI Client Growth Engine, helping users:

retain clients

expand accounts

upsell services

generate referrals

Instead of the typical CRM model:

rep waits for client problems

Momentum should behave like:

AI detects opportunity → prompts rep to act.

Critical Layout Requirement

The Client Pipeline workspace must match the Sales Pipeline workspace layout exactly.

Use the same 3-column structure visible in the screenshot.

Column layout

Left column
Client context and operational controls

Centre column
Account intelligence and insights

Far right column
AI panel (AI Sales Engine / AI Client Growth Engine)

The AI panel must remain on the far right, exactly like the Sales Pipeline.

Do not move the AI panel into the centre.
Do not create a different layout pattern for clients.

This must feel like the same product surface, not a new page type.

Product Concept

The existing AI Sales Engine helps win deals.

Inside the Client Pipeline, the same framework becomes a Client Growth Engine.

Sales workflow:

prospect → qualify → close

Client workflow:

retain → expand → upsell → referrals

The system should help the rep identify:

growth opportunities

retention risks

expansion opportunities

referral opportunities

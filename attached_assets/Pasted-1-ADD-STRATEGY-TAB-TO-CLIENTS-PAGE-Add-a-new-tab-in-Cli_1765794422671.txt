1) ADD STRATEGY TAB TO CLIENTS PAGE

Add a new tab in Clients → Client Details named Strategy with 4 sub-tabs:

Overview (Gap View)

Plan (30/60/90)

Insights (Assets + Keywords + Competitors)

Content (Drafts + Approvals)

This Strategy section must be usable without any other pages.

2) CLIENT BUSINESS PROFILE (WHAT + WHERE) IS REQUIRED CONTEXT

Add a persistent businessProfile object to:

orgs/{orgId}/clients/{clientId}

Fields:

businessProfile: {
  industry: "",
  primaryServices: [],
  secondaryServices: [],
  primaryLocations: [],
  secondaryLocations: [],
  serviceAreaType: "local" | "regional" | "multi-location",
  idealJobType: "",
  averageJobValue: null,
  seasonalityNotes: null
}


Rules:

No strategy, content, or NBA can be generated without referencing businessProfile.

Any update to businessProfile triggers a full strategy refresh + new Action Queue items.

3) STRATEGY KICKOFF WIZARD (MINIMUM USER INPUT, NO THINKING)

In Clients → Strategy, add a button:

“Start / Refresh Strategy”

Wizard must be 6 steps max, designed to be completed in 2–4 minutes:

Confirm industry (prefill from businessProfile if exists)

Confirm primary + secondary services (multi-select + add custom)

Confirm primary + secondary locations (multi-select + add suburbs)

Choose ONE primary goal:

Map Pack (Top 3)

More Calls/Leads

Organic Rankings

Lower CPL / Better Lead Quality

Optional URLs (no blocking validation):

websiteUrl

gbpUrl (or placeId)

facebookUrl

instagramUrl

“What’s working / not working” (multi-select + optional notes)

On submit:

Save answers into a strategySession record (see section 6)

Immediately queue an AI strategy generation job (server-side)

4) AUTO-STRATEGY CREATION (NO CLIENT LEFT WITHOUT STRATEGY)

If client.strategyStatus is null OR no active strategyPlan exists:

Within 24 hours of client creation (or immediately on Deal→Won conversion), AI must:

Create a baseline Strategy Plan

Create Action Queue items from that plan

Write Client History: “Baseline strategy auto-generated”

5) ASSET ANALYSIS (WEBSITE + GBP + SOCIAL) OPTIONAL BUT AUTOMATIC IF URLS PROVIDED

Add an Insights engine that runs server-side when URLs exist:

Website: fetch HTML + key pages, extract titles/headings/body, detect service/location coverage and conversion elements

GBP: fetch key profile signals (category/services/photos/posts frequency if available; otherwise store “unknown”)

Social: fetch recent public posts only (frequency + themes; no private data)

Store results under:
orgs/{orgId}/clients/{clientId}/insights/{insightId}

Each insight write must also create a History event:
“Assets analysed: website/gbp/social”

Cache insights:

Website insights cache 7 days

GBP insights cache 3 days

Social insights cache 7 days

7) KEYWORD DEMAND + KEYWORD GAP (API-DRIVEN, STORED, REUSABLE)

Add optional “Market Research” inside Insights:

orgs/{orgId}/clients/{clientId}/marketResearch/{reportId}

It must support two modes:

Google-first mode: Search Console + Keyword Planner (if connected)

External provider mode (configurable): DataForSEO (if keys exist)

Outputs to store:

keywordClusters (service + location combos)

volumes (range acceptable if Google returns ranges)

intent labels (commercial/local/info)

detected gaps:

“high demand but no page/supporting content”

“wrong intent covered”

“missing suburb/service coverage”

Every “gap” must generate at least one Strategy initiative and at least one Action Queue item.

Cache marketResearch 14–30 days.

Inside marketResearch, add competitor mapping:

Local competitor set from Places/GBP category + radius (or from user-provided competitors)

Compare:

review count vs client

posting cadence signals (if available)

category/service completeness signals (if available)

Store as:

competitorList [{ name, placeId, websiteUrl, signals }]

competitorBenchmarks used to calculate targetState and gapSummary

reason + whyBullets

10) AI CLIENT GROWTH ORCHESTRATOR (AUTONOMOUS STRATEGY MAINTENANCE)

Add a continuous server-side orchestrator that runs without the user initiating it:

Schedules:

Daily: check for missing strategy, overdue deliverables, upcoming review dates

Weekly: strategy drift detection + content cadence checks

Monthly: refresh market research if enabled and cache expired

Responsibilities:

Ensure every client always has an active strategy plan

Detect strategy drift (activity completion vs plan, deliverable stalls, declining health trend)

Regenerate strategy when drift detected (or flag “needs_review” and create a corrective Action Queue call)

Generate content drafts automatically when cadence requires it (see section 11)

Always push the next best execution steps into Action Queue

The user’s role is approval and execution, not planning.

11) AUTONOMOUS CONTENT CADENCE (DRAFTS + APPROVAL QUEUE)

Create:
orgs/{orgId}/clients/{clientId}/contentDrafts/{draftId}

Draft types:

seoBlog

gbpPost

facebookPost

landingPageOutline

reviewRequestTemplates

Cadence rules (examples; configurable per product/cadenceTier):

SEO product and no blog drafted in 30 days → generate blog draft

GBP product and no GBP post drafted in 14 days → generate GBP post draft

New target keyword cluster added → generate supporting content draft

Every draft must:

be stored in DB

write Client History (“Draft generated: type”)

create Action Queue item: “Review and approve draft”

Approval actions:

Approve → mark draft approved, log Activity, create “publish” task if needed

Reject → require reason, suppress regeneration 7 days

“What’s next” (top 3 actions from Action Queue)

it has tooltips (desktop hover + mobile tap)

it displays “Biggest gap” sentence below the chart

Chart axes must be client-readable (max 6):

GBP Strength

Reviews & Trust

Local Coverage (services/locations)

Website Conversion

Content Momentum

Competitive Position
